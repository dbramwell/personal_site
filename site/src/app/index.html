Current and Past Work

FDM: Novemeber 2011 – Present

FDM is an IT services provider with a large number of clients on a global scale. While working with FDM I was placed on two client sites in two different roles:

UBS Investment Bank (Dev-Ops): April 2013 - Present

Implemented a “build once” pipeline for 9 major components using gradle and teamcity. During this process we migrated from using maven as a build tool to using gradle to free ourselves from the maven build cycle, and configured our teamcity pipeline to pass built artifacts along to avoid rebuilding in the various test phases. Developed a build radiator using Groovy, Ratpack, AngularJS and Javascript to display current build statuses, as well as a score board of which Developers had broken builds or commited on broken builds without fixing them. Performed a migration of data from 10g to 11g Oracle databases in two test environments before decommissioning the 10g databases. Coordinated an upgrade of 3 testing environments from using an MQ6 to an MQ7 server, making changes to our own environment and liasing with other teams to make changes to their connected applications. Developed scripts using Groovy, gradle, and teamcity, rally, github and svn APIs to automate processes between the tools such as creating changesets in Rally for each commit, creating teamcity projects and scm roots for new repos, reporting on rally stories included in the next release and more. Created a server that adds dynamic images to pull requests that report on the status of the teamcity pipeline for that branch as well as master, to enable devs to see easily if it's ok to merge.

Betfair (QA automation engineer): November 2011 – March 2013

Developed a number of test packs for front-end applications and APIs using java, groovy and selenium. Implemented a solution using Jenkins and maven to build and run test packs daily against production to monitor status, as well as against newly deployed code in the various pre-prod environments as part of the continuous delivery build pipeline. Provided testing support for changes in third party applications that could possibly affect our own, as well as an upgrade from Oracle 10 to Oracle 11 and a migration of the production data centre.
